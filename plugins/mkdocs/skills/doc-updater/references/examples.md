# doc-updater スキル: 使用例

このファイルでは、doc-updaterスキルの具体的な使用例を示します。

## 例1: 週次ドキュメント更新（推奨パターン）

**ユーザーリクエスト:**
```
今週の変更をREADMEに反映して
```

**実行手順:**

1. export_diff.sh -week -e "docs/" を実行
2. ai_context.diff を読み込み
3. 新機能、変更点を抽出
4. README.mdを更新
5. 報告

**期待される成果:**
- 今週追加された新機能がREADMEに記載される
- 変更されたAPIや設定がREADMEに反映される
- ドキュメントとコードの乖離が解消される

## 例2: 特定機能のドキュメント追加

**ユーザーリクエスト:**
```
認証機能のドキュメントをREADMEとSPECに追加
```

**実行手順:**

1. export_diff.sh -week src/auth/ -e "*.test.ts" を実行
2. ai_context.diff を読み込み
3. 認証関連の変更を抽出
4. README.mdに使用例を追加
5. docs/SPEC.mdにAPI仕様を追加
6. 報告

**期待される成果:**
- 認証機能の使用例がREADMEに追加される
- 認証APIの詳細仕様がdocs/SPEC.mdに追加される
- セキュリティ関連の注意事項が記載される

## 例3: リリース前のドキュメント総合更新

**ユーザーリクエスト:**
```
リリース前にすべてのドキュメントを最新化
```

**実行手順:**

1. export_diff.sh -month -e "docs/" を実行（1ヶ月分の変更）
2. ai_context.diff を読み込み
3. すべての変更を分類（新機能、バグ修正、破壊的変更など）
4. README.md、docs/SPEC.md、docs/guides/ を更新
5. CHANGELOG.md を生成（オプション）
6. 報告

**期待される成果:**
- すべてのドキュメントが最新状態に更新される
- 破壊的変更が明確に記載される
- リリースノートやCHANGELOGが生成される（必要に応じて）

## 使用のヒント

### 対象期間の選択

- **日次更新**: `-day` または `-day 3` で直近の変更のみ対象
- **週次更新**: `-week` で過去1週間の変更を対象（推奨）
- **月次更新**: `-month` でリリース前の総合更新に使用

### 除外パターンの活用

- テストファイルを除外: `-e "*.test.ts" -e "*.spec.ts"`
- ビルド出力を除外: `-e "dist/" -e "build/" -e "out/"`
- ドキュメント自体の変更を除外: `-e "docs/"`

### 対象ディレクトリの絞り込み

- 特定機能のみ: `src/auth/` `src/api/` など
- フロントエンドのみ: `components/` `app/` など
- バックエンドのみ: `server/` `api/` など
