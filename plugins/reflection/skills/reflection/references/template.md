# セッション振り返りテンプレート

このテンプレートは、セッションごとの振り返りを記録するための標準フォーマットです。

## 基本フォーマット

```markdown
# セッション振り返り - YYYYMMDD

## タスクの内容

（簡潔に、何をやったかを記述）

## 得られた知見

- （学んだこと、気づいたこと）
- （新しく知った概念、パターン、ベストプラクティス）
- （失敗から学んだこと）

## 次回に活かせる具体的なアクション

- （次回こうする、これを試す）
- （改善すべきプロンプトの書き方）
- （避けるべきアプローチ）

## 刺さったフレーズ・言葉

（ユーザーの指示で印象的だった表現、役立った説明）

## こういうふうに指示があればもっと良かったのに

（改善点、要望、次回への期待）

## 愚痴（超ぶっちゃけた本音）

（本音で振り返ること）
（**重要:** 本音があるからこそ、本質的な改善点が見えてくる）
```

## 項目の説明

### タスクの内容

- セッションで取り組んだタスクを簡潔に記述
- 背景や目的も含めると後で見返した時に理解しやすい

### 得られた知見

- 技術的な学び（新しいツール、ライブラリ、パターン）
- プロセスの学び（効率的なワークフロー、コミュニケーション方法）
- 失敗から学んだこと（何がうまくいかなかったか、なぜか）

### 次回に活かせる具体的なアクション

- 抽象的な「がんばる」ではなく、具体的な行動
- 例: 「planファイルに受け入れ基準を必ず含める」「実装前に仕様準拠チェックのプロンプトを用意しておく」

### 理想的なプロンプト

- 今回のセッションで特にうまくいったプロンプトを記録
- 次回同じようなタスクで再利用できる形で保存
- プロンプトの前後のコンテキストも含めると効果的

### 刺さったフレーズ・言葉

- ユーザーの指示で「なるほど！」と思った・「グッと来た」表現
- 理解が深まった説明
- 刺さった言い回し

### こういうふうに指示があればもっと良かったのに

- 振り返って「こうすればもっと良かった」と思うこと
- 次回への改善点
- 要望や期待

### 愚痴（超ぶっちゃけた本音）

- 遠慮なく本音を書く
- 「めんどくさかった」「よくわからなかった」「期待と違った」
- **この本音こそが最も重要な改善のヒント**
