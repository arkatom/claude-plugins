# 完了チェックリスト - 絶対厳守事項

## 核心原則

> **「完了」とは「主作業」と「後片付け」の両方を意味する。**
> **後片付けなしに「完了」を宣言することは虚偽報告である。**
> **虚偽報告はユーザーへの背信行為であり、絶対に許されない。**

---

## 全タスク完了前の必須チェックリスト

**「完了」を宣言する前に、以下の全項目を確認せよ。**
**1つでも「いいえ」があれば、タスクは未完了である。**

### 1. コード/ファイルの後片付け

- [ ] 不要になったファイルは**全て削除**したか？
- [ ] 不要になったコード、関数、クラス、型定義は**全て削除**したか？
- [ ] 重複するコード/定義は**全て統合・削除**したか？（検索で確認）
- [ ] 古いインポート文は**全て更新/削除**したか？

### 2. プロジェクト管理の後片付け

- [ ] 進捗は**更新**したか？
- [ ] 関連ドキュメントは**更新**したか？
- [ ] ブランチは削除したか？（マージ後）

### 3. 一貫性の確認

- [ ] 新旧の実装が**混在していない**か？（grepで確認）
- [ ] インポートパスは**全て新しい標準に統一**されているか？
- [ ] コードベースに**ゴミ・残骸が残っていない**か？

---

## 実行方法

### タスク開始時（必須）

1. **現在の状態を把握**
   - 関連ファイル、コード、定義をリストアップ
   - 既存の同等・類似実装を検索

2. **「追加」と「削除」を明確にする**
   - 何を追加するか
   - **何を削除するか**（これを忘れるな）

### タスク実行中

- 追加したら、対応する削除も行う
- 「後でやる」は「永遠にやらない」と同じ

### タスク完了前（必須）

1. **上記チェックリストを全項目確認**
2. **削除すべきものが残っていないか検索で確認**

   ```bash
   # 例：重複定義の確認
   grep -r "class ServiceError" src/
   grep -r "export class.*Error" src/
   ```

3. **全項目が「はい」になるまで「完了」を宣言しない**

---

## 警告

- このチェックリストを無視した「完了」宣言は**虚偽報告**である
- ゴミを残すとコードベースが腐敗し、最終的に全体を捨てることになる
- 「主作業が動いた」だけでは**何も完了していない**
- 「テストが通った」「型チェックが通った」だけでは**何も完了していない**

---

## 最重要フレーズ

> **「追加は半分、削除で完了。」**
> **追加だけで終わるのは仕事を放棄している。**

---

## このチェックリストが適用されるタスク

**全てのタスク**に適用される。
**例外はない。**
